NODE JS

Node.js is a cross-platform, open-source runtime environment for the server layer.We're going to use this to run JavaScript code on the server. I've decided to use the latest version of Node, v6.3.0 at the time of writing, so that we'll have access to most of the new features introduced in ES6.

EJS

"E" is for "effective." EJS is a simple templating language that lets you generate HTML markup with plain JavaScript. No religiousness about how to organize things. No reinvention of iteration and control-flow. It's just plain JavaScript.

EXPRESS

As per their website, Express is a "Fast, unopinionated, minimalist web framework for Node.js", that we're going to be building our Todo list application on.

MONGOOSE

Mongoose provides a straight-forward, schema-based solution to model your application data. It includes built-in type casting, validation, query building, business logic hooks and more, out of the box.

Simple, minimalist and fast.

MODELS

We will have 2 models: User and Articule. The first has include: id and password, second include: title, creator, content, time and image.

ROUTES

We will use express to define our routes. This will be included in our controller. The routes is: write, login, register, article and create.

SERVER

We create our server with Node. Remember to define the port you will use. We use 3000 ;^)

DATABASE

We connect through MongoDB using mongoose and promises, we manage CRUD with mongoose. Parameters are handled in config.js

DINAMIC VIEW

Like a lot of the applications we build, there will be a lot of code that is reused. We'll call those partials and define three files we'll use across all of our site: head.ejs and footer.ejs. Let's make those files now.

STRUCTURE OF PROJECT

We have divided the project in a way that is easier to understand: config / includes parameters of the bd and the port, libs / carries inside the controller, models / includes the models that we define for our blog, public / includes the public access files dah ..., views / where we use the templates of ejsy finally app.js which is the conglomerate of all this
LET'S SEE HOW THE MAGICS HAPPENS ...

The structure of the project is designed so that everything is in a natural direction, in this way the code is simpler to understand and avoids confusion for the most novice.
